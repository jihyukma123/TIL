# 질문 목록

Q: 프로젝트 관련 - 업무 지침이 공유되는 case가 있음(메일이나, 텍스트로 공유된다고 함) 이런 것들을 프로젝트에 계속 add하면, 검색했을 때 바로바로 확인이 가능한지?
A: 지침이 제공되는 형식이? -> 메일로 보통 제공됨. 파일을 40개 정도 올릴 수 있음. 그런 내용이 있을 때마다 올릴 수 있음 다만 이전꺼하고 충돌이 될 수 있음. 그래서 파일 현행화가 필요함
-> 이어서 비슷한 질문: Custom GPT 를 학습 시켜서 쓸 수 있는데, 플젝이랑 GPT 차이점? 올리는 파일 개수 제한? Custom GPT는 학습 없이 답변하고, 프로젝트는 학습해서 답변하는게 맞는지? 파일의 업로드나 chat의 개수 제한이 있는지?
-> A: 파일 올릴 수 있는 한도는 각각 정해져있음. 프로젝트랑 GPT 차이는? -> 프로젝트는 100명까지 초대가능하기 때문에 논의를 활발하게 하거나, 할 때 사용하는 걸 권고드림. 거기서 결정된게 있으면 custom GPT로 하면 좋을 것 같다. 논의하는 방이 있을수도 있고, 아니면 프로젝트 자체를 누적 발전하는 GPT 처럼 사용하고 싶으면 프로젝트를 사용, 프로젝트는 재배포가 필요없다. 협업 관점에서는 프로젝트가 나을 수도 있다. 둘 다 엄밀히 말해서 '학습'은 아니다. GPT는 개인적인 궁금증에 한해서 사용하는거고, 플젝은 다른 사람의 대화/궁금증도 같이 공유해서 볼 수 있다.

## [질문 목록]

프로젝트에 업무 지침(메일 등)을 계속 추가하면 검색 시 바로 확인 가능한가요?
→ 파일 업로드는 가능하나, 이전 자료와 충돌 가능성이 있어 현행화 관리 필요
→ 파일 개수 한도 존재

프로젝트와 Custom GPT의 차이는 무엇인가요?
→ 프로젝트는 최대 100명 협업 가능, 대화 공유 중심
→ Custom GPT는 특정 기능을 배포하는 형태
→ 둘 다 ‘학습’이 아니라 컨텍스트 기반 응답 구조

Custom GPT 업데이트 시 이전 버전은 유지되나요?
→ 업데이트하면 즉시 전체 사용자에게 반영됨 (이전 버전 유지되지 않음)

Custom GPT 공유 방식 차이는 무엇인가요?
→ 사용자 직접 초대: 인원 한정
→ 링크 공유: 링크 보유 시 접근 가능
→ 그룹 단위 초대는 현재 미지원

프로젝트 내 질문은 얼마나 쌓이나요?
→ 대화는 지속 누적됨
→ 중요한 내용은 계정 기반으로 저장되어 이후 대화에 활용됨

## [특이사항]

‘학습’이라는 표현은 정확하지 않으며,
대화 이력 중 중요하다고 판단되는 내용이 저장되어 이후 응답 시 함께 입력되는 구조임.

Excel 기반 자동화 니즈가 매우 높으며,
데이터가 AI가 처리하기 쉬운 형태로 정제되지 않으면 할루시네이션 발생 가능성이 있음.

→ 분기별 데이터 업데이트 체계 필요
→ AI가 처리하기 쉬운 구조로 변환하는 전처리 봇 도입 검토 필요

구매팀 내 이미 3개 Agent를 사용 중이며,
1개 Agent에 대한 고도화 니즈가 존재함.
