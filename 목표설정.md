# 목표 설정이 필요하다.

공부를 해야된다는 생각은 많고, 앉아는 있는데 뭘 해야되는지 명확하게 정해져있지 않아 시간을 낭비하는 문제가 있음.

큰 목표를 설정해야지 그로부터 작은 목표들을 추출해서, 매일매일 무엇을 공부할지 계획을 세울 수 있음.

`목표와 방향성이 없는 어설픈 꾸준함`을 극복해보자.

# 큰 목표

- 개발이 좋다. 재밌음.
- 사람들이 마주한 문제를 해결하는 역할을 하고 싶음. -> 왜
  - 타인을 돕는 과정에서 기쁨과 보람을 느꼈음. 인정을 받는 것보다 그냥 타인을 돕는 것이 좋다고 생각하고, 딱히 대가를 바라지 않고 그냥 도울 수 있기 때문에 돕는 성향임.

> 기술적 이해를 바탕으로, 사람들의 문제를 가장 '본질적이고 지속 가능하게' 해결하는 엔지니어

그리고 이왕이면, 자신있게 내가 이 분야에 대해서는 잘 안다라고 말할 수 있는 전문가가 되고 싶음.
-> 이 분야가 뭐임? 프로그래밍

# 목표 쪼개기

가장 기본적으로는 기술적 이해가 필요함.

- 문제 해결을 위한 프로덕트를 구현하기 위해서 사용하는 도구에 대한 이해
  - IDE
  - 개발 언어(JS/TS, Python, Java/Kotlin)
  - AI를 활용한 개발 사이클 및 효율 개선 방법
  - Database
  - Framework
    - 회사에서 사용하는 표준 프레임워크에 대한 이해(React, Spring Boot, FastAPI)
- 문제 해결을 위한 프로덕트가 동작하는 환경에 대한 이해
  - 직접 환경
    - 웹
  - 간접 환경
    - 네트워크
    - 운영체제
    - 클라우드
    - 형상관리
- 문제 해결에 필요한 구조 이해
  - 아키텍처
  - FE, BE 등 각 요소의 역할과 어떻게 서로 상호작용 하는지
  - 트레이드오프에 대한 이해
