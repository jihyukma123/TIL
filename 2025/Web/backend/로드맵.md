# 백엔드 개발을 하기 위해서 갖춰야할 최소한의 지식들

영상을 보고 정리가 되는 느낌이라서 올해 하반기에 공부할 내용들을 정리해봤다.

참고로 백엔드의 3요소는 Web, WAS, DB라고 한다.

# 1. API 개발 및 설계 능력

1. 게시판 CRUD

- REST API 설계
  - HTTP 메서드(GET, POST, PUT, DELETE 등)
  - 중요한 것은 REST를 준수하는 API를 설계하는 것. 어떻게 REST를 준수하면서 API를 설계할 것인지에 대한 고민이 필요함
  - HTTP 메서드 각각
  - 처리된 상태에 따른 code
- 인증/인가
  - 세션/쿠키, JWT
  - 게시글 권한 처리
- 에러 처리
  - 적절한 HTTP 상태 코드
- API 문서화
  - Swagger, OpenAPI

# 2. 데이터베이스

3요소 중 하나

참고로 DB는 관계형 DB를 공부해야 한다.(NoSQL은 나중에...)

- MYSQL, PostgreSQL
- ERD
  - 테이블 간 관계인 ERD 설계하고 설명할 수 있어야 함.
- SQL
  - 쿼리 짤 줄 알아야 함. ORM만 쓰다보니 쿼리 작성이 안되는 경우가 많음
  - ORM이 생성하는 쿼리를 볼 것
  - DB에서 어떤 raw query로 실행되는지?
  - 조인과 서브쿼리 이해
- 인덱스
  - 아무데서나 사용하면 안됨
  - WHERE 조건 보면서 필요한 곳에 적절하게 사용하는 것이 필요

# 3. 클라우드 배포

실제 사용자가 접속할 수 있는 환경에 배포하는 것

- 배포해보기
  - EC2, RDS 사용(특히 RDS도 꼭 사용해보는 것을 추천)

# 4. 전반적인 웹의 흐름

유저 -> 브라우저 -> WEB -> WAS -> DB

WEB인 nginx이런 쪽을 내가 진짜 아예 모름.

예를 들어서, 로그인 요청 같은 경우에는 nginx config에 WAS로 넘겨라 라는 설정이 있다고 함(WEB에서 처리하기에는 복잡한 작업이라고 하는 듯.)

WAS -> Application Server(Python의 uvicorn)가 nginx의 http요청을 받아서, application(python의 fastapi)가 이해할 수 있는 형태로 변환을 해서 전달

Application이 필요한 처리를 DB와 연계해서 작업해서 거꾸로 다시 데이터를 반환

# 위의 최소 기준에 미달인 경우....어떻게 Gap을 메꾸는가??

1. REST API를 잘 모르겠다.

그러면 게시판 CRUD연습을 해라.

- 게시글 작성
- 로그인/로그아웃 등등

처음에는 기간 고려하지 않고 완성하고, 익숙해지면 2일(16시간) 내로 개발하는 연습. DB는 무조건 MySQL 이나 postgresql

2. DB를 잘 모르겠다?

- ORM이 실행하는 쿼리 출력해보기
- 단일 쿼리만 실행하는게 아니라, 조인을 최소 한 번 해서 실행해보기.
- 전광철 OCP 유튜브 보기 [주소](https://www.youtube.com/@ocp396/videos)

3. 클라우드 배포 잘 모르겠다.

- 개발한 게시판 배포
- AWS EC2 + RDS 사용
- 인그레스 설정(인바운드)
- 익숙해지면 VPC 설정까지.
  - EC2, RDS를 private subnet으로 설정하고, Gateway이용해서 EC2만 외부에서 접근가능하도록 설정
- 실습 완료 후 자원삭제하는 것 잊지 않기.

4. 전반적인 웹의 흐름 잘 모르겠다?

- 널널한 개발자 네트워크 핵심이론 기초 강의 보기
- 36강(그림 한 장으로 외워서 끝내는 웹 서비스 구조 기본이론)은 특히 꼭 보도록 한다.

지금의 상태는 매우 좋습니다. 이미 FastAPI로 실전 경험을 해봤다는 것은, 이론만 아는 것보다 훨씬 큰 자산입니다. 부족한 기본 지식은 그 경험 위에 쌓아 올리면 되기 때문에, 매우 효율적인 학습 로드맵을 짤 수 있습니다.

9월부터 11월까지 3개월 동안 진행할 백엔드 기본기 강화 로드맵을 다음과 같이 제안합니다.

1단계 (9월): 기본기 다지기 - 데이터와 통신
백엔드의 가장 핵심인 **'데이터를 어떻게 다루고 전달할지'**에 대한 기본 지식을 집중적으로 쌓는 기간입니다.

1. 데이터베이스(DB) 기본기:

목표: ORM만 쓰던 수준에서 벗어나, DB의 원리를 이해하고 SQL을 능숙하게 다룬다.

학습 내용:

SQL 문법: SELECT, INSERT, UPDATE, DELETE 등 기본적인 CRUD 쿼리를 자유롭게 작성하는 연습을 합니다. JOIN을 활용해 여러 테이블을 엮는 방법을 익히는 것이 중요합니다.

관계형 데이터베이스(RDB)의 개념: 테이블, 스키마, 인덱스, 기본 키/외래 키의 역할 등을 이해합니다.

데이터 정규화(Normalization): 왜 데이터를 정규화하는지, 1차, 2차, 3차 정규형이 무엇인지 개념을 익힙니다.

실천 방안: 간단한 PostgreSQL DB를 Docker로 띄워보고, FastAPI 프로젝트와 연동하여 ORM 없이 직접 SQL 쿼리를 날려보는 연습을 합니다.

2. 네트워크 기본기:

목표: API 통신이 어떻게 이루어지는지 HTTP의 작동 원리를 이해한다.

학습 내용:

HTTP 프로토콜: GET, POST, PUT, DELETE 등의 메서드와 HTTP 상태 코드(Status Code), 헤더(Header)의 의미를 정확히 파악합니다.

RESTful API 디자인 원칙: 'RESTful'이 왜 중요하고, 어떤 규칙을 따르는지 깊이 있게 공부합니다.

실천 방안: Insomnia나 Postman 같은 API 클라이언트를 사용해 다양한 HTTP 메서드로 당신의 FastAPI 서버에 직접 요청을 보내보며 테스트합니다.

2단계 (10월): 설계와 구조 - 확장 가능한 코드
개별적인 지식들을 조합해 '어떻게 하면 좋은 구조의 백엔드를 만들 수 있을지' 고민하는 단계입니다.

1. 백엔드 아키텍처:

목표: 시스템을 설계할 때 고려해야 할 큰 그림을 그린다.

학습 내용:

모놀리스(Monolith) vs. 마이크로서비스(Microservices): 두 아키텍처의 장단점을 비교하고, 어떤 상황에서 적합한지 파악합니다.

레이어드 아키텍처: 프레젠테이션, 비즈니스 로직, 데이터 접근 레이어를 나누는 개념을 이해합니다.

클린 아키텍처(Clean Architecture): 코드를 더 견고하고 유연하게 만드는 설계 원칙을 개략적으로 알아봅니다.

실천 방안: 기존에 만들어봤던 FastAPI 프로젝트를 리팩토링(refactoring)하여 레이어를 나눠보세요. (예: routers, services, repositories 폴더로 분리)

2. 보안과 인증:

목표: API를 안전하게 만들고, 사용자를 식별하는 방법을 이해한다.

학습 내용:

인증(Authentication) vs. 인가(Authorization): 두 개념의 차이를 명확히 구분합니다.

JWT(JSON Web Token): JWT가 무엇이고, 어떤 방식으로 사용자 인증에 활용되는지 원리를 이해합니다.

OAuth 2.0: 소셜 로그인 등에서 사용되는 인증 프로토콜의 작동 원리를 간단히 이해합니다.

3단계 (11월): 시스템 운영과 확장성
실제 서비스가 **'대량의 트래픽을 안정적으로 처리하려면 어떻게 해야 할지'**에 대한 지식을 쌓는 단계입니다.

1. 시스템 확장성(Scalability):

목표: 트래픽 증가에 대비하여 시스템을 확장하는 방법을 이해한다.

학습 내용:

수직 확장(Vertical Scaling) vs. 수평 확장(Horizontal Scaling): 두 방식의 차이점과 장단점을 이해합니다.

로드 밸런서(Load Balancer): 여러 서버로 트래픽을 분산하는 로드 밸런서의 개념을 알아봅니다.

캐싱(Caching): DB 부하를 줄이기 위해 데이터를 임시 저장하는 캐시의 필요성과 원리를 이해합니다. (예: Redis의 역할)

2. 컨테이너와 배포:

목표: 개발 환경과 실제 서비스 환경을 통일하고, 배포 과정을 이해한다.

학습 내용:

Docker 기본: Docker가 왜 필요한지, Dockerfile을 작성하는 방법, docker-compose를 활용하는 방법을 익힙니다.

클라우드 서비스(AWS) 맛보기: AWS의 주요 서비스 중 EC2와 RDS가 어떤 역할을 하는지 간단히 알아봅니다.

실천 방안: 기존의 FastAPI 프로젝트를 Docker 이미지로 만들고, 로컬에서 여러 컨테이너를 띄워보며 로드 밸런서의 개념을 체감해 봅니다.

이 로드맵은 당신이 이미 가진 실무 경험 위에 견고한 이론적 기반을 쌓아, 다음 단계로 나아가기 위한 훌륭한 디딤돌이 될 것입니다. 이 과정을 통해 얻은 지식을 사내 문서로 정리하고 공유한다면, 당신의 노력과 역량을 확실하게 증명할 수 있을 것입니다.
