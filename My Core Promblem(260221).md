## 한 문장 진단

**나는 "공부하는 사람"이 아니라 "공부하고 싶어하는 사람"이다.**

---

## 핵심 문제 구조

### 1. 인과관계가 뒤집혀 있었다

내가 그린 인과 체인:

> 목표 불명확 → 뭘 할지 모름 → 이것저것 건드림 → 깊이 없음

실제 인과 체인:

> **불편함을 견디는 능력 부족** → 어려워지면 이탈 → 아무것도 완주 못함 → 자신감 하락 → 목표가 흐려짐 → "목표가 없어서 그런가보다" → 또 새 목표 설정 → 반복

목표는 이미 충분히 세웠다. SICP, Composing Software, CMU DB, AWS CCP — 전부 "뭘 해야 될지 몰라서" 못한 게 아니라, **알고 시작했는데 중간에 그만둔 것**이다. "집중력/끈기 부족"이 부수 요인이 아니라 **근본 원인**이었다. 더 정확하게 말하면, 불편함을 견디는 능력이 부족하다는 점

### 2. 메타학습의 무한루프

이 저장소에서 가장 많은 비중을 차지하는 건 기술 학습이 아니라 **"어떻게 공부할지에 대한 고민"**이다.

목표설정.md → 시스템초안.md → 회고.md → 핵심 역량.md → AI한테 평가 요청 → ...

목표를 세우고, 시스템을 만들고, 회고를 하고, 다시 목표를 세우는 루프. 이건 학습이 아니라 **학습 불안을 관리하는 행위**이고, 실제 학습을 회피하면서 생산적인 느낌을 주는 **지연(procrastination)의 고급 형태**다.

### 3. 수집 ≠ 학습

312개 파일 중 306개가 마크다운. 코드가 거의 없다. 대부분이 외부 자료의 요약, 링크 모음, 키워드 나열 — Pocket 북마크와 다를 바 없는 것들. "접했다"는 것과 "이해했다"는 것 사이의 간극을 좁히지 못했다. AI가 써준 긴 글을 그대로 붙여넣는 패턴도 반복됐는데, 읽을 때만 그럴듯하고 기억·전이·실전 대응에는 거의 안 남는다.

### 4. 넓이만 있고 깊이가 없다

React Native, React, JS, TS, Python, FastAPI, AWS, AI/LLM, 딥러닝, 함수형 프로그래밍, 시스템 디자인, DB, SICP, Kotlin, Java, Spring Boot... 이건 공부 목록이 아니라 **불안 목록**이다. 기본기가 빈약한 상태에서 트렌드만 좇으면 "말만 번지르르한 개발자"가 된다.

### 5. 구현 근육의 퇴화

AI가 코드를 짜주는 환경에 기대서, 직접 바닥부터 짜보고 디버깅하는 고통을 회피했다. "AI가 다 짜주니까 문제 정의가 핵심"이라는 건 기초가 탄탄한 사람한테나 해당되는 말이다.

## 그래서 뭘 해야 하는가

| # | 행동 | 이유 |
|---|------|------|
| 1 | **메타학습을 중단한다** | 더 이상의 자기 분석, 목표 설정, 시스템 설계는 학습이 아니라 회피다. 이미 충분히 알고 있다. |
| 2 | **하나를 끝낸다** | 시작만 하고 중단된 시리즈가 최소 8개. 새로운 걸 시작하기 전에 하나를 완주한다. 완주 경험이 다음 완주를 만든다. |
| 3 | **코드를 친다** | 글 정리는 학습의 보조 수단이지 학습 자체가 아니다. 손이 기억하지 못하는 지식은 진짜 지식이 아니다. |
| 4 | **주제를 2개로 줄인다** | 다 하려다 아무것도 못하는 패턴을 끊는다. 나머지는 필요해질 때 한다. |
| 5 | **기록 기준을 바꾼다** | "오늘 뭔가를 접했다" → 기록 (X). "오늘 직접 해보고 이해했다" → 기록 (O). 검증하지 않은 것은 기록하지 않는다. |
| 6 | **어려운 지점에서 탭을 닫지 않는다** | 이해가 안 되는 그 순간이 학습이 시작되는 지점이다. 정확히 거기서 매번 이탈했다. |

## 한 줄 결론

**원인은 이미 알고 있다. 더 분석할 건 없다. 남은 건 하나를 골라서, 어려운 부분이 나왔을 때 탭을 닫지 않는 것뿐이다.**
