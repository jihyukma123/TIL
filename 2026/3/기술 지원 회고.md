# ChatGPT Enterprise Reselling 기술지원 회고

## Detail이 중요하다 

보다 '전문적'으로 느껴지는 support를 제공하려면 작은 Detail에서 더 신경쓰는게 필요함

예를 들어서, 

- Compliance API를 별도 모니터링 용 tool에 연동하기 전까지 데이터는 어떻게 관리할 것인지?
  - Guidance를 먼저 제공하는게 좋아보임
- 처음에 고객한테 ChatGPT 도입을 위해서 어떤 작업이 필요한지에 대해서 Overview 제안이 필요
  - 이러이러한 작업이 필요하다. 그래서 우리가 이런 점에서 가이드를 줄거고, 권장사항도 제공할거다.
- 고객 입장에서 IT 담당자가 해야될 일/정책담당자가 해야될 일/계약 담당자가 해야될 일/보안 담당자가 해야될일 등등을 교통정리를 미리 해주는게 좋음
 
## 기술적 지식과 이해가 필요함

3가지 항목에 대해 깊이 있는 지식 필요

- 네트워크 -> header injection, SASE 등에 대한 이해 필요
  - Personal Workspace 접근을 막기 위해서 어떤 처리를 할 수 있는지에 대한 이해 필요
- 보안 -> TLS, AES 뿐만 아니라, enterprise에서 요구되는 보안 구조나 tool에 대한 이해가 필요함(DLP, SIEM 등)
  - ChatGPT 연동 시 고객들이 어떤 보안적 선택을 할 수 있는지에 대한 이해 필요
- 인증 -> SSO, OIDC, SAML 개념 잘 알아야 함.
  - Okta 등 잘 알려진 서비스를 사용하지 않는 경우에 대해 대비 필요

iteration & improve가 되어야 하고, 누가 해도 대응이 일관되게 처리될 수 있도록 구성하는게 필요

## 적절한 소통 방식?

- 내 입장에서 적절한 소통과, 고객의 입장을 고려한 소통은 다를 수 있음.
    - 나는 전화가 더 적절하다는 생각이 드는 순간들이 있었는데, 메일로 수신한 문의에 대해서는 바로 처리가 어렵더라도 일단 메일로 처리하는게 낫다는 피드백을 받았다(from Wifey..)
    - 전화는 진짜 급한 상황에서만 사용하라고..
  - 전화 -> 내가 편한 소통 방식이었던 것 같음. 어 이거는 그냥 내가 구두로 설명하는게 더 편할 것 같은데? 싶어서 전화로 설명한 적도 있고, 고객이 요청한 사항에 대해서 기술적인 추가 검토가 필요할 것 같아서 이에 대해 시간이 더 필요한 점을 고객에게 설명하려고 전화하기도 함.
  - 근데 고객의 입장은? 이 일만 하는게 아님. 즉 내가 전화로 갑자기 들이닥쳐서 '아까 보낸 메일 설명드릴려고요~~' 하면 마냥 달갑지 않은 상황일 수 있다.
    - 더 나은 소통방식은
      - 일단 메일로 잘 답변한다 timely 하게 답변이 어려운 경우, 이유를 서면으로 설명하는게 맞음.
        - 고객이 추가 설명을 요구하는 경우 전화로 하는게 나음.
      - 전화가 필요한 경우는 정말 급한 경우, 아니면 고객이 요청한 경우 이렇게 나눠서 하는게 좋은 것 같음.
      - 메일이 지금 상황에서 더 나은 추가적인 이유는 히스토리가 남는다는 점. 그리고 공유받는 사람들이 진행상황을 알 수 있다는 점.
