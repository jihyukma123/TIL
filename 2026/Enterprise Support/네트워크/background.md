# [AGENT CONTEXT] 네트워크 폴더 배경

> AI Agent가 이 폴더의 문서들을 읽을 때 참조하는 컨텍스트 문서.

---

## 작성자 역할

**ChatGPT Enterprise 기술 지원 담당자 (OpenAI 측)**

고객사가 자사 네트워크 환경에서 ChatGPT Enterprise를 운영할 때 발생하는 네트워크/보안 관련 기술 문제를 진단하고 안내하는 것이 주요 업무다.

---

## 이 폴더의 목적

업무에서 마주치는 기술 개념들의 **배경 지식**을 쌓기 위한 폴더다.

ChatGPT Enterprise의 기능 설명서나 구현 가이드가 아니라, 그 기능들이 어떤 기술 원리 위에서 동작하는지를 이해하기 위한 **기초 기술 학습 노트**다.

예를 들어 고객 지원 업무에서 "Header Injection"이나 "SSL Inspection"이라는 단어가 나왔을 때, 그게 무엇인지 원리 수준에서 설명할 수 있는 것을 목표로 한다.

---

## 학습 순서 및 개념 범위

앞의 개념을 알아야 뒤의 개념이 이해되는 구조로 나열한다.

### 1단계 — 네트워크 통신의 기초

- **OSI 7 Layer 모델** — 네트워크 통신이 어떤 계층 구조로 이루어지는지. 이후 나오는 모든 개념의 좌표계가 된다.
- **HTTP / HTTPS** — 요청과 응답의 구조, 헤더란 무엇인지. L7(애플리케이션 계층)에서 일어나는 일.
- **SSL/TLS** — 암호화 통신의 원리, 인증서가 왜 필요한지. HTTPS가 HTTP와 무엇이 다른지.

### 2단계 — 엔터프라이즈 네트워크 보안 장비

기업이 내부 네트워크와 인터넷 사이의 트래픽을 통제하기 위해 어떤 장비/구조를 쓰는지.

- **방화벽(Firewall)** — L3/L4 수준에서 트래픽을 허용/차단. Flow-based vs Proxy-based 차이.
- **VPN** — 원격 사용자가 기업 네트워크에 안전하게 접속하는 원리.
- **SWG (Secure Web Gateway)** — L7 수준에서 웹 트래픽을 검사하고 통제하는 장비. 방화벽보다 더 깊이 들여다본다.
- **CASB (Cloud Access Security Broker)** — SaaS 서비스 접근을 가시화하고 통제하는 레이어.
- **Zero Trust / SASE** — "내부망이라고 믿지 않는다"는 현대 보안 아키텍처의 방향성. 위 장비들이 왜 이 방향으로 진화했는지.

### 3단계 — 장비가 트래픽에 개입하는 방법

2단계에서 배운 장비(SWG 등)가 실제로 트래픽을 어떻게 열어보고 조작하는지.

- **SSL Inspection** — 암호화된 HTTPS 트래픽을 중간에서 복호화해 내용을 확인하는 원리. SSL/TLS와 인증서 개념이 전제가 된다.
- **Header Injection** — 복호화한 HTTP 요청에 헤더를 강제로 삽입하는 원리. HTTP 헤더 개념과 SSL Inspection이 전제가 된다.

### 4단계 — 실제 솔루션

- Zscaler, FortiGate 등이 위 개념들을 실제로 어떻게 구현하는지.

## Agent가 알아야 할 컨텍스트

- 작성자는 개발자이지만, 네트워크 개념들에 대해서 foundation이 탄탄하지 않다.
- 문서는 "이 개념이 업무에서 왜 등장하는가"라는 맥락과 함께 작성된다.
- 단순 암기가 아니라 원리를 이해하는 것이 목표다.
